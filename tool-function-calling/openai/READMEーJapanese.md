## 💰 アカウント要件と費用についての注意

この記事では OpenAI API を使用します。

コードを実行する前に、次のものが準備できていることを確認してください。

プログラムから API リクエストを送るたびに、トークンがリモートの AI サービスへ送信されます。これらのリクエストには実際の費用が発生し、API アカウントに請求される可能性があります。

自動化を実行する前に、次の安全策で自分を守りましょう。

### 厳しめの利用上限を設定する

学習中は、プロバイダーの請求ダッシュボードにログインし、**$5.00** などの低めの日次または月次予算上限を設定してください。

これにより、暴走したスクリプトがクレジットを使い切ったり、予期しない請求を発生させたりするのを防ぎやすくなります。

### サンドボックスのループでは高価なモデルを避ける

単純なテスト、デバッグ、CI ループでは、可能な場合は低コストのモデルを使用してください。

より大きいモデルや高価なモデルは、対象を絞った本番用途のために取っておきましょう。

### ループを保護する

これらのスクリプトを使って独自の自動化を書く場合は、必ず厳格なループ上限を設定してください。

たとえば次のようにします。

```ts
for (let i = 0; i < 5; i++) {
  // 安全な回数制限付きループ
}
```

これにより、無限の非同期ループが API エンドポイントを呼び続けてしまうのを防ぎやすくなります。

## OpenAI API による関数呼び出し / ツール使用

この記事では、次の大きな概念に進みます。

```text
モデルは、私たちのプログラムに関数の実行を依頼できる。
```

これは通常、**関数呼び出し** または **ツール使用** と呼ばれます。

この記事では、OpenAI API による関数呼び出しに特化して説明します。

今後の記事では、同様の例を Gemini API と Claude API でも扱います。

## 関数呼び出しとは何か

関数呼び出しとは、AI モデルがひそかに自分でコードを実行するという意味ではありません。

代わりに、私たちのアプリケーションが実行できる 1 つ以上の関数をモデルに説明します。するとモデルは、ユーザーのリクエストに答えるために、そのうちの 1 つが役立つと判断できます。

モデルは、構造化された関数呼び出しリクエストを返します。

私たちの TypeScript コードはそのリクエストを読み取り、実際の関数を実行し、その結果をモデルに返します。

簡略化すると、流れは次のようになります。

```text
ユーザーが質問する
        ↓
OpenAI がリクエストを受け取り、利用可能なツールを見る
        ↓
OpenAI がツール呼び出しを要求する
        ↓
私たちの TypeScript コードが関数を実行する
        ↓
関数の結果を OpenAI に返す
        ↓
OpenAI が最終的な応答を書く
```

重要な境界線は次のとおりです。

```text
モデルは関数呼び出しを要求する。
アプリケーションが関数を実行する。
```

この境界線が重要なのは、アプリケーション側が制御を持ち続けられるからです。

モデルは、データベース、ファイル、メール、内部システムに直接アクセスするわけではありません。モデルが要求できるのは、アプリケーションが明示的に利用可能にしたツールだけです。

## 関数呼び出しが重要な理由

通常の AI レスポンスは、モデルに文章を書かせたり、要約させたり、何かを説明させたり、分類させたり、テキストを変換させたりしたい場合に役立ちます。

構造化出力は、モデルに予測しやすいデータを返してほしい場合に役立ちます。

関数呼び出しは、モデルがアプリケーションの助けを必要とする場合に役立ちます。

たとえば、モデルは次のようなことを必要とするかもしれません。

* 顧客レコードを検索する
* 予約可能時間を確認する
* 注文状況を取得する
* 商品カタログを検索する
* サポートチケットを作成する
* 価格を計算する
* アプリケーションコードから授業アクティビティを取得する

モデルは、ユーザーのリクエストを理解するのが得意です。

アプリケーションは、実際のコードを安全に実行するのが得意です。

この 2 つを組み合わせることで、より実用的なワークフローを作れます。

```text
モデルがリクエストを理解する。
アプリケーションが処理を実行する。
モデルが結果を説明する。
```

## より現実的な例

小規模ビジネス向けのメールアシスタントを想像してみましょう。

顧客が次のメッセージを送ってきます。

```text
こんにちは。配管工事の予約を変更したいです。金曜の午前に来てもらう予定でしたが、来週のどこかに変更したいです。
```

基本的な AI スクリプトでも、丁寧な返信を書くことはできます。

それは便利ですが、できることは限られています。

関数呼び出しのワークフローなら、さらに多くのことができます。

```text
顧客メールを読む
        ↓
顧客を検索する関数を呼び出す
        ↓
現在の予約を調べる関数を呼び出す
        ↓
来週の空き時間を確認する関数を呼び出す
        ↓
実際に空いている候補日時を含む返信案を作成する
        ↓
人間の承認用に下書きを送る
```

このようなシステムでは、モデルが業務アプリケーションを置き換えているわけではありません。

モデルは、どのアプリケーション関数を使う必要があるかを判断する手助けをしています。

実際の作業は、引き続きあなたのコードによって制御されます。

この記事では、そのような完全なシステムは作りません。

今回は、関数呼び出しのパターンそのものに集中できるように、とても小さなローカル TypeScript 関数を使います。

## この記事で使う例の関数

この記事では、授業用の理科アクティビティを返す小さな関数を作成します。

この関数は 2 つの引数を受け取ります。

* `topic`
* `gradeLevel`

`topic` 引数は、アクティビティのテーマを関数に伝えます。

`gradeLevel` 引数は、アクティビティをどのくらい簡単または詳細にするかを変えます。

この例は小さいですが、主な考え方を示せます。

```text
モデルが引数を渡す。
関数がその引数を使う。
引数に応じて結果が変わる。
```

## 共有関数ファイルを作成する

このファイルを作成します。

```text
src/science-activity.ts
```

次のコードをファイルに追加します。

```ts
export type GradeLevel = "elementary" | "middle-school";

export type ScienceActivityInput = {
  topic: string;
  gradeLevel: GradeLevel;
};

export type ScienceActivity = {
  topic: string;
  gradeLevel: GradeLevel;
  title: string;
  materials: string[];
  instructions: string;
};

export function getScienceActivity(
  input: ScienceActivityInput
): ScienceActivity {
  const isElementary = input.gradeLevel === "elementary";

  return {
    topic: input.topic,
    gradeLevel: input.gradeLevel,
    title: isElementary
      ? `${input.topic} の観察アクティビティ`
      : `${input.topic} の調査アクティビティ`,
    materials: isElementary
      ? ["紙", "鉛筆", "色鉛筆"]
      : ["理科ノート", "鉛筆", "タイマー"],
    instructions: isElementary
      ? `${input.topic} について気づいたことを描き、それについて一文を書きましょう。`
      : `${input.topic} を観察し、証拠を記録し、どのようなパターンに気づいたかを説明しましょう。`,
  };
}
```

この関数は意図的にシンプルにしています。

データベースは呼び出しません。

別の API も呼び出しません。

それ自体では AI を使いません。

これは普通の TypeScript コードです。

この点が重要です。関数呼び出しは、モデルに魔法のようにコードを実行させるものではないからです。

関数呼び出しとは、私たちのアプリケーションに特定の関数を実行するよう、モデルが依頼できる仕組みです。

## この関数が引数をどのように使うか

この関数は、渡された両方の引数を使います。

`topic` の値は、アクティビティのタイトルと説明文に表示されます。

たとえば topic が `光合成` の場合、関数は光合成のアクティビティに関するテキストを返せます。

`gradeLevel` の値は、アクティビティの内容を変えます。

`elementary` の場合、関数はより簡単な観察アクティビティを返します。

`middle-school` の場合、関数は少し詳しい調査アクティビティを返します。

つまり、コードで次のように呼び出すと、

```ts
getScienceActivity({
  topic: "光合成",
  gradeLevel: "elementary",
});
```

関数はより簡単なアクティビティを返します。

しかし、次のように呼び出すと、

```ts
getScienceActivity({
  topic: "光合成",
  gradeLevel: "middle-school",
});
```

関数は少し詳しいバージョンを返します。

これは中核となる考え方を示すには十分です。

モデルがどの引数を送るかを決め、TypeScript 関数がその引数を使います。

## なぜこの関数を別ファイルに置くのか

この関数を専用ファイルに置くことで、API 呼び出し用のコードを OpenAI に集中させられます。

あとで Gemini や Claude の同様の記事を作成する場合、それらの例でも同じ関数を使えます。

プロジェクトは次のようになります。

```text
src/
├── science-activity.ts
└── openai-tool-call.ts
```

OpenAI スクリプトは、`science-activity.ts` から関数をインポートします。

これにより、OpenAI ファイルは次の内容に集中できるため、例がすっきりします。

* OpenAI 用のツールを定義する
* リクエストを送信する
* 関数呼び出しを読み取る
* ローカル関数を実行する
* 関数の結果を返す
* 最終レスポンスを表示する

## なぜインポートで `.js` を使うのか

別の TypeScript ファイルからこの関数をインポートする場合、インポートは次のようになります。

```ts
import {
  getScienceActivity,
  type ScienceActivityInput,
} from "./science-activity.js";
```

作成したファイル名は次のため、少し不思議に見えるかもしれません。

```text
science-activity.ts
```

しかし、このプロジェクトでは現代的な Node.js ECMAScript Modules を使っています。

この設定では、TypeScript のソースファイルは `.ts` を使いますが、インポートパスは TypeScript が JavaScript になった後に Node.js が期待する形で書きます。

つまり、ソースファイルは次の名前ですが、

```text
science-activity.ts
```

インポートパスでは次のように書きます。

```text
./science-activity.js
```

これは、`"module": "nodenext"` と `"type": "module"` を使う場合の一般的なパターンです。

## OpenAI 関数呼び出しの例

`src/science-activity.ts` に共有関数を作成できたので、次は OpenAI 専用のスクリプトを書きます。

このスクリプトでは次のことを行います。

* ローカルの `getScienceActivity` 関数を OpenAI のツールとして説明する
* ユーザーリクエストを OpenAI API に送る
* モデルから関数呼び出しリクエストを受け取る
* ローカル TypeScript 関数を実行する
* 関数の結果を OpenAI に返す
* ユーザー向けの最終レスポンスを表示する

このファイルを作成します。

```text
src/openai-tool-call.ts
```

次のコードをファイルに追加します。

```ts
import OpenAI from "openai";

import {
  getScienceActivity,
  type ScienceActivityInput,
} from "./science-activity.js";

const client = new OpenAI();

const tools = [
  {
    type: "function" as const,
    name: "getScienceActivity",
    description:
      "リクエストされた理科のテーマと学年レベルに合う授業用アクティビティを取得します。",
    parameters: {
      type: "object",
      properties: {
        topic: {
          type: "string",
          description:
            "授業アクティビティの理科テーマ。例: 光合成、重力。",
        },
        gradeLevel: {
          type: "string",
          enum: ["elementary", "middle-school"],
          description:
            "生徒の学年レベル。より簡単なアクティビティには elementary、少し詳しいアクティビティには middle-school を使います。",
        },
      },
      required: ["topic", "gradeLevel"],
      additionalProperties: false,
    },
    strict: true,
  },
];

function parseScienceActivityInput(argumentsJson: string): ScienceActivityInput {
  const parsed = JSON.parse(argumentsJson) as Partial<ScienceActivityInput>;

  if (typeof parsed.topic !== "string") {
    throw new Error("ツール引数 'topic' は文字列である必要があります。");
  }

  if (
    parsed.gradeLevel !== "elementary" &&
    parsed.gradeLevel !== "middle-school"
  ) {
    throw new Error(
      "ツール引数 'gradeLevel' は 'elementary' または 'middle-school' である必要があります。"
    );
  }

  return {
    topic: parsed.topic,
    gradeLevel: parsed.gradeLevel,
  };
}

async function main() {
  try {
    const firstResponse = await client.responses.create({
      model: "gpt-5.5",
      instructions:
        "あなたは親切な授業アシスタントです。ユーザーが授業用の理科アクティビティを求めた場合は、利用可能なツールを使ってください。",
      tools,
      tool_choice: {
        type: "function",
        name: "getScienceActivity",
      },
      input:
        "私は小学校で理科を教えています。光合成についての授業用アクティビティをください。",
    });

    const functionCall = firstResponse.output.find(
      (item) => item.type === "function_call"
    );

    if (!functionCall) {
      console.log("OpenAI レスポンス:");
      console.log(firstResponse.output_text);
      return;
    }

    if (functionCall.name !== "getScienceActivity") {
      throw new Error(`不明なツールが要求されました: ${functionCall.name}`);
    }

    const activityInput = parseScienceActivityInput(functionCall.arguments);
    const activity = getScienceActivity(activityInput);

    const finalResponse = await client.responses.create({
      model: "gpt-5.5",
      previous_response_id: firstResponse.id,
      instructions:
        "先生向けの親しみやすい最終回答を書いてください。ツール結果を使いますが、JSON、関数呼び出し、内部的なツール実行については言及しないでください。",
      input: [
        {
          type: "function_call_output",
          call_id: functionCall.call_id,
          output: JSON.stringify(activity),
        },
      ],
    });

    console.log("OpenAI 最終レスポンス:");
    console.log(finalResponse.output_text);
  } catch (error) {
    console.error("OpenAI 実行エラー:", error);
  }
}

main();
```

## OpenAI スクリプトを `package.json` に追加する

`package.json` ファイルを開き、`"scripts"` ブロックを探します。

次のエントリを追加します。

```json
"start:openai-tool-call": "tsx src/openai-tool-call.ts"
```

`"scripts"` ブロックには、これまでの記事で追加したエントリがすでにあるかもしれません。それで問題ありません。

最後のスクリプト行以外には、各行の末尾にカンマが必要です。

## OpenAI ツール呼び出しスクリプトを実行する

プロジェクトのルートからスクリプトを実行します。

```bash
npm run start:openai-tool-call
```

## 私のレスポンス

あなたの結果はおそらく異なりますが、私が受け取ったレスポンス例は次のとおりです。

```text
OpenAI 最終レスポンス:
もちろんです。小学生向けに、観察と表現を中心にしたシンプルな光合成の授業アクティビティを紹介します。

## アクティビティ名  
**「植物はどうやって育つの？ 光合成を絵で表そう」**

## ねらい  
植物が、**光・水・空気**を使って自分で栄養をつくっていることに気づく。

## 準備物  
- 紙  
- 鉛筆  
- 色鉛筆  

## 進め方  

### 1. 導入  
子どもたちに次のように問いかけます。

「植物はごはんを食べないのに、どうして大きくなるのでしょう？」

児童の考えを自由に出させます。  
「水を飲むから」「太陽の光を浴びるから」などの意見を板書するとよいです。

### 2. 光合成について簡単に説明  
小学生向けには、次のように伝えると分かりやすいです。

「植物は、太陽の光を使って、水と空気の中の成分から、自分の栄養をつくります。これを光合成といいます。」

必要に応じて、黒板に  
**太陽の光 ＋ 水 ＋ 空気 → 植物の栄養**
```

## OpenAI が関数呼び出しをどのように扱うか

OpenAI Responses API では、関数は **ツール** の一種としてモデルに提供されます。

ツール定義は、モデルに次のことを伝えます。

* 関数名
* 関数が何をするか
* どの引数を受け取るか
* どの引数が必須か

このスクリプトでは、ツール定義は次のように始まります。

```ts
const tools = [
  {
    type: "function" as const,
    name: "getScienceActivity",
    description:
      "リクエストされた理科のテーマと学年レベルに合う授業用アクティビティを取得します。",
```

重要なのは、これが関数を実行しているわけではないという点です。

これは、`getScienceActivity` という関数が利用可能であることを OpenAI に伝えているだけです。

次に、関数が受け取る引数を説明します。

```ts
parameters: {
  type: "object",
  properties: {
    topic: {
      type: "string",
      description:
        "授業アクティビティの理科テーマ。例: 光合成、重力。",
    },
    gradeLevel: {
      type: "string",
      enum: ["elementary", "middle-school"],
      description:
        "生徒の学年レベル。より簡単なアクティビティには elementary、少し詳しいアクティビティには middle-school を使います。",
    },
  },
  required: ["topic", "gradeLevel"],
  additionalProperties: false,
},
strict: true,
```

これは、前回の記事で扱った構造化出力の考え方に似ています。

違いは、このスキーマがツール呼び出しの引数を説明している点です。

つまり、モデルにレッスンカードを直接返してもらうのではありません。

モデルには次のように伝えています。

```text
授業用の理科アクティビティが必要な場合は、getScienceActivity の呼び出しを要求してよい。
その呼び出しを要求するときは、topic と gradeLevel を指定する。
```

この学習用の例では、`tool_choice` も使います。

```ts
tool_choice: {
  type: "function",
  name: "getScienceActivity",
},
```

デフォルトでは、モデルは 0 個、1 個、または複数のツールを呼び出すかどうかを判断できます。

この記事では、関数呼び出しを明確に示すため、ツール呼び出しが確実に起きるようにしています。

次に、最初のリクエストを送信します。

```ts
const firstResponse = await client.responses.create({
  model: "gpt-5.5",
  instructions:
    "あなたは親切な授業アシスタントです。ユーザーが授業用の理科アクティビティを求めた場合は、利用可能なツールを使ってください。",
  tools,
  tool_choice: {
    type: "function",
    name: "getScienceActivity",
  },
  input:
    "私は小学校で理科を教えています。光合成についての授業用アクティビティをください。",
});
```

このリクエストは、モデルに次のものを渡します。

* ユーザーのリクエスト
* ツール定義
* ツールをいつ使うべきかを伝える instructions
* このデモ用に特定のツール呼び出しを強制する `tool_choice` 設定

モデルがツールを使うと、OpenAI は次の型を持つ出力アイテムを返します。

```ts
"function_call"
```

このスクリプトでは、レスポンス出力からその関数呼び出しを検索します。

```ts
const functionCall = firstResponse.output.find(
  (item) => item.type === "function_call"
);
```

関数呼び出しには、関数名と、モデルが渡したい引数が含まれます。

その引数を信頼する前に、コードで解析して確認します。

```ts
const activityInput = parseScienceActivityInput(functionCall.arguments);
```

次に、TypeScript コードが実際の関数を実行します。

```ts
const activity = getScienceActivity(activityInput);
```

ここが重要な瞬間です。

OpenAI が `getScienceActivity` を実行したわけではありません。

モデルが関数呼び出しを要求しました。

アプリケーションが関数を実行しました。

関数の実行後、その結果を OpenAI に返します。

```ts
const finalResponse = await client.responses.create({
  model: "gpt-5.5",
  previous_response_id: firstResponse.id,
  instructions:
    "先生向けの親しみやすい最終回答を書いてください。ツール結果を使いますが、JSON、関数呼び出し、内部的なツール実行については言及しないでください。",
  input: [
    {
      type: "function_call_output",
      call_id: functionCall.call_id,
      output: JSON.stringify(activity),
    },
  ],
});
```

`function_call_output` アイテムには、ローカル TypeScript 関数から得られた結果が入ります。

`call_id` は、その結果を OpenAI が要求した特定のツール呼び出しに結びつけます。

`previous_response_id` は、この 2 回目のリクエストが 1 回目のレスポンスから続いていることを OpenAI に伝えます。

この時点で、モデルは次のものを持っています。

* 元のユーザーリクエスト
* モデル自身が要求した関数呼び出し
* 私たちのコードが返した関数結果

これで、モデルは最終回答を書けます。

```ts
console.log(finalResponse.output_text);
```

OpenAI の関数呼び出しの流れは次のようになります。

```text
ツールを定義する
        ↓
ツールを利用可能にしてユーザーリクエストを送る
        ↓
OpenAI から function_call を受け取る
        ↓
引数を解析して検証する
        ↓
ローカル TypeScript 関数を実行する
        ↓
function_call_output を OpenAI に返す
        ↓
最終的なモデルレスポンスを表示する
```

主な学びは次のとおりです。

```text
OpenAI は、モデルが関数呼び出しを要求できるようにする。
実際の関数実行は、引き続きアプリケーションが制御する。
```

だからこそ、関数呼び出しは強力です。

モデルはツールが必要かどうかを判断できますが、どのツールが存在するか、そのツールが何をできるか、その結果をどう扱うかは、コード側が制御します。

## 学んだこと

この記事では、最小限の OpenAI 関数呼び出しの例を作成しました。

重要なのは、具体的な授業アクティビティそのものではありません。

重要なのは次のパターンです。

```text
関数をモデルに説明する。
モデルに関数呼び出しを要求させる。
自分のコードで関数を実行する。
結果をモデルに返す。
モデルを使って最終回答を生成する。
```

これは、基本的なプロンプトとレスポンスのスクリプトから大きく進んだステップです。

基本的なスクリプトでは、モデルが自分の学習内容と現在のプロンプトコンテキストから回答します。

関数呼び出しのスクリプトでは、モデルがアプリケーションに助けを求めることができます。

## 例を小さく保った理由

この記事の例では、データベース、API、外部サービスではなく、ローカルの TypeScript 関数を使いました。

これは意図的です。

目的は、まだ完全な本番システムを作ることではありません。

余計な要素を増やさずに、関数呼び出しのパターンをはっきり理解することが目的です。

このパターンが理解できたら、ローカル関数を、次のようなより現実的なものに置き換えることができます。

```text
getCustomerByEmail(...)
getAvailableAppointments(...)
getOrderStatus(...)
searchKnowledgeBase(...)
createEmailDraft(...)
```

OpenAI API の流れは似たものになります。

主な違いは、アプリケーション関数が裏側で何をするかです。

## 関数呼び出しはエージェントの構成要素

関数呼び出しは、AI エージェントを支える構成要素の 1 つでもあります。

エージェントは通常、1 つの関数を 1 回呼び出すだけではありません。

より高度なエージェントでは、次のような流れになることがあります。

```text
ユーザーリクエストを読む
        ↓
ツールを選ぶ
        ↓
結果を確認する
        ↓
別のツールを選ぶ
        ↓
その結果を確認する
        ↓
回答できるまで続ける
```

この記事では、そのような完全なループは作りません。

しかし、この記事では、そのようなワークフローを可能にする中核的な仕組みを紹介しました。

エージェントを作る前に、まずはより単純な形を理解することが大切です。

```text
1 つのリクエスト
1 つのツール呼び出し
1 つの関数結果
1 つの最終回答
```

## 重要な安全上の注意

関数呼び出しは、モデルがアプリケーションにアクションを要求するための方法を提供します。

そのため、どのツールを公開するかには注意が必要です。

初心者向けの例では、小さなローカル関数を呼び出すだけなので安全です。

実際のアプリケーションでは、次の点を慎重に考える必要があります。

* 引数を検証する
* 各ツールができることを制限する
* ツール呼び出しをログに記録する
* 機密性の高い操作には承認を必要にする
* 破壊的な操作が自動実行されないようにする
* 予期しないモデル出力を処理する
* 個人情報や機密データを保護する

モデルが構造化された引数を提供した場合でも、重要な処理を実行する前に、コード側で必ず検証するべきです。
